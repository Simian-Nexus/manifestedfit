The plugin exists at 03_Website/wordpress/plugins/manifested-fit-content-engine/ — three files:

manifested-fit-content-engine.php — the whole engine. A "Content Engine" admin page with settings (API key, Telegram, model), a topic queue you paste topics into, a Run Now button, a Send Telegram test button, and an activity log. The runner picks the day's persona (your full schedule is baked in — Mon Frankie/Tue Dana/Wed Nadia/Thu Dana/Fri Rowan, weekends random, 2026 Canadian holidays → Frankie, and Sep 30 / Nov 11 → a short respectful Rowan piece with no CTA), takes the next queued topic, calls the Claude API with structured JSON output (so the title/slug/excerpt/categories/HTML always parse cleanly), creates the post as a draft under the persona's byline — resolved by login name, so the missing Dana/Nadia user IDs no longer matter — sets the RankMath focus keyword, and pings your Telegram with the edit link. Failures (empty queue, API error) also ping Telegram, so silence = nothing to review.
cron-runner.php — a PHP-CLI fallback for Bluehost cron in case the curl/REST route ever hits shared-hosting timeouts (the AI call takes 1–3 minutes).
README.md — install steps, the exact cron command (the admin page also displays it with your secret filled in), and cost notes (~$0.07/post on Opus, ~$2/month daily).
